# sdd-template.md — 软件设计规格说明书（SDD）模板

> 由 D5 检查点或 /mims sdd 命令加载，基于 domain-model.yaml 生成 sdd.md

## 生成原则

偏结构化描述但使用业务语言。SDD 的价值在于补充 YAML 无法表达的业务含义：
- 对象和关系的业务含义（不是逐字段展开属性列表）
- 状态流转的业务解释（不是逐条列出转移条件）
- 规则的业务场景示例（不是复制 constraint 表达式）

**摘要引用而非逐字段镜像**：
- 对象属性表格只列 chinese_name + type + required + 一句说明，不展开 data_dictionary/validations
- 状态表格只列 chinese_name + is_initial/is_final
- 关系表格只列 from + label + to + 一句说明
- 属性详情直接查阅 `domain-model.yaml`

**模型引用规则**：SDD 中出现模块、对象、状态、转移、操作、规则时，表格或小节标题必须保留模型 id（如 mod_001、obj_001、st_001、tr_001、op_001、rule_001），以便从文档反查 `domain-model.yaml`。
